Understanding kleptographic attacks

This guide provides a more in-depth explanation of the key generation attacks offered by the org.jcryptool.visual.kleptography plug-in. Although kleptographic attacks can be designed for any asymmetric cryptographic algorithm, this plug-in is based on RSA. It is assumed that the user is familiar with the mathematics behind RSA encryption. The ideas and algorithms used in the plug-in are largely based on those from the thorough and unique book [YY04]. That book is highly recommended for an even more in-depth analysis of kleptography, as well as cryptovirology in general. A FAQ for cryptovirology is offered by the authors of the book at [Yo06].

Kleptography is "the study of stealing information securely and subliminally", as defined by the authors of [Yo06], who in fact coined the word. The idea is to introduce an asymmetric backdoor into a black-box cryptosystem such that an implementation of this system will appear to function normally for all users except the individual attacker that can use some special information to aid in decrypting some or all of the generated key pairs. Even if the black box device were to be opened and investigated, it would only be possible to notice that an attack is underway, but it would still be impossible to make use of the backdoor without the attacker's special information.

The authors of [YY04] describe an attack on RSA that meets these criteria. They call it a secretly embedded trapdoor with universal protection (SETUP) attack. This plug-in includes

  • a slightly simplified version of the SETUP attack,
  • three other, less secure attacks, and
  • honest key generation (for comparison).

Each of these attacks assume either that the manufacturer is dishonest and has introduced some sort of weakness or backdoor into the given cryptographic device, or that an outside attacker has somehow infected the device and altered it to suit his or her needs.

For more information about these key generation methods and their implementations in this plug-in, please refer to the pages for each specific algorithm:

  • Getting started: honest key generation
  • Using a fixed prime P
  • Generating P via a pseudo-random function
  • Generating P via a pseudo-random generator
  • SETUP attack

The purpose of demonstrating these attacks in a contained environment is to indicate that it is essentially impossible to know that your keys are entirely secure. Even the strictest statistical tests may not reveal the presence of a well-hidden backdoor. You must be able to completely trust the manufacturer of the cryptosystem implementation and also trust that it has not been attacked and altered to contain a vulnerability.

[YY04] A. Young, M. Yung, Malicious Cryptography: Exposing Cryptovirology, John Wiley & Sons, 2004.

[Yo06] A. Young, Cryptovirology FAQ, Version 1.31, http://www.cryptovirology.com/.
