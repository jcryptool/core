X.509 and PKIX

In this section, you get detailled information about the two most important standards for digital certificates on the internet. The ITU-T recommendation X.509 (abbreviated as X.509 standard or X.509) is a standard for public key and privilege management infrastructures. X.509 specifies only the format for certificates and certificate revocation lists. The main implementation of X.509 and the thus the second most important standard for digital certificates RFC5280 is authored by a workgroup of the IETF and is entitled "Internet X.509 Public Key Infrastructure Certificate and Certificate Revocation List Profile". This standard is referred to as PKIX standard. In most cases if an X.509 certificate is mentioned, it refers to the X.509v3 certificate from RFC5280.

X.509v3 certificate

RFC5280 specifies by this time the third version of the X.509 certificate profile. A certificate in accord with X.509v3 contains the following fields.

  • Version – The used X.509 version (1, 2 or 3)
  • Serial number – An assigned serial number by the CA, which has to be unique within the CA.
  • Signature Algorithm – Signature and hash algorithms, which can be used with this certificate, for example MD5 with RSA (md5WithRSAEncryption)
  • Issuer – The assigning CA, consisting of
    • Common Name – The Common Name is the name, to whom the certificate is issued. This name can also be an email address or a server name if it is an e-mail certificate or server certificate.
    • Organisation – Name of organisation, which run the PKI.
    • Organisational Unit – Department of organisation.
    • Location – The place where the CA is located. Mostly, the city where the operator has its headquarters.
    • State – The state where the CA is located.
    • Country – The country where the CA is located.
  • Validity – The validity period of the certificate. During this period, the CA guarantees to manage the validity status of the certificate. This consists of a "not before" and a "not after" the date.
  • Subject – Information about the certificate owner. These consist of the same elements as the information about the CA, but some fields can be empty. At least, the common name must be set.
  • Subject Public Key Info – The pubilc key of the certificate owner.
  • X.509v3 extensions – Optionale extensions. For example a root certificate is labled with a X.509v3 extensions.
  • Signature – Signature algorithm and signature of the assigning CA over the hash of the certificate.
