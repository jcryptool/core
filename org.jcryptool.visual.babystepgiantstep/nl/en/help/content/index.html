Shanks Babystep Giantstep Algorithm

The demonstration shows the functioning of the Shanks Babystep Giantstep Algorithm plug-in.

Shanks Babystep Giantstep Algorithm plug-in could be started by the menu Visualization or via Crypto-Explorer tab Visualization and maximizes his view.

The view of plug-in contains 6 sections: The first section describes the algorithm in its execution. The explanations adjust corresponding to the current step.

The second section contains the user inputs. The values can be entered manually or they can be chosen through the given parameters via the dropdown menus.

By clicking on the Continue to step 2 button will be check the validity of the inputs. Even if one or more of them are invalid a pop-up dialog will open to correct the entries. The invalid values are marked with a red background-color.

In section 3 executes the calculations of the group order, the upper bound of the order and the multiplicative inverse..

In the next step, section 4, the babysteps are calculated and presented in a table with all the iterations.

By clicking on the button Continue to step 4 the giantsteps will be calculated. The giantsteps are also presented in a table. Depending on the inputs is it possible that this step is skipped because there is no need to calculate the giantsteps.

The button Compute result will be enabled and by clicking on this button the result will be shown in the section Result. If we find a solution for the DL-problem the values of the solution are marked in the tables with the same color as in the Result section.

The result is shown in the section Result in color green. In this section the calculation formula is also displayed and the output equation.

If there is no solution for the discrete logarithm then a message is displayed.

The button Reset clears all user inputs, the calculations of the result and also clears the contents of the tables and the algorithm can be started again for new user inputs.
