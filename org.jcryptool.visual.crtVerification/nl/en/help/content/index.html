Certificate Verification

Welcome to the online help of the plug-in which validates certificates according to the validity models:

  • shell model
  • modified shell model
  • and chain model.

Below you find an overview of the plug-in and a more detailed description of the usage of the plug-in. This plug-in helps to demonstrate the validity of digital signatures using different certificate trees, consisting of a Root CA, CA and a User CA, and validity models. It's also possible to change the period of validity of each certificate as well as the verification and signature date with sliders. After this modification you can validate this information. But you have to consider that these changes are just a simulation. That means: No certificates were changed and there were also no new certificates created.

At the end is a short description of the validity models.

Usage of the plug-in

Below you find how to use the plug-in on the basis of an example scenario.

Step 1

The first step is to load the certificates. If you click on the buttons "Load Root CA", "Load CA" and "Load User Certificate" a dialog window opens and you can choose the certificate which you want to load out of the JCT-Keystore. After you have chosen the right certificate you have to click on "finish". The sliders in the Graphical User Interface are now in the right position and the font colour of the buttons is now green.

 

Step 2

If all three certificates (Root CA, CA and User CA) were loaded successfully you can choose the validity model with the radio buttons. Now you can press the "Validate" button. If the symbol next to this button is an X in a red circle the validation was unsuccessful if the symbol is a checkmark in a green circle the validation was successful.

The log shows you all the information, which leads to this result.

Step 3

There is the possibility to change the position of the sliders to simulate what would happen if the dates change. But you have to consider that these changes are just a simulation. That means: No certificates were changed and there were also no new certificates created. The verification slider can only be manipulated in the shell model because it's the only model where the verification date is needed for the validation. It's possible to adjust the months of a date with the sliders. If you want to set the days too you can do this with the text fields in "Details". After the modification of a slider the font colour of the buttons "Load Root CA", "Load CA" and "Load User Certificate" changes to orange.

Validity models

Validity models describe, which requirements have to be fullfilled at a certain point of time to consider a signature as valid. These models are the shell model, a modified version of the shell model and the chain model.

Shell model

When using the shell model, a key(-pair) is valid at a certain time, if the corresponding certification path is also valid at this time. The certification path is valid, if all contained certificates are valid at this time.

A certificate is valid, if

  • the signature of the certificate is valid,
  • the regarded time is within the validity period of the certificate,
  • when the check is performed, the certificate is not on the issuning CA's Certificate Revocation List or if the certificate is on this list, the revocation time is later than the time of the ckeck

Modified shell model

The modified shell model is, with the exception of a special condition, equivalent to the shell model which is described above. The difference is, that when the check of a certification path is performed, the signing date of a document is used.

Chain model

In the chain model, the only requirement is, that each certificate had to be valid at the moment it was used to create a signature. This means that: at the time of the signation of a document, the client's certificate had to be valid, at the time when the clients's certificate was issued, the issueings CA's certificate had to be valid and so on. This model does not consider if a certificate was revoked since the signation time.
